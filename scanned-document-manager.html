Scanned Document Manager

 

Click to download Scanned Document Manager Guide

 

The OnePractice Scanned Document Manager is a useful tool providing the means to greatly reduce the time spent by your staff filing scanned documents into SharePoint.

 

The application combines state-of-the-art technologies to intelligently read, categorize and file scans from most common scanners and organize them in SharePoint according to options you have configured.

 

When we first considering the requirements of accounting practices, we look to all aspects of the practices operation.  One thing we found was taking a lot of time whether the practice was using an electronic document management system or not with filing documents correctly, specifically those that came in through the post or delivered on paper.

 

Looking around at solutions in the market, we discovered that although there were some solutions which worked for specific forms.  There was no solution which would take any documents, no matter how complex and automatically file it based on the client.

 

After just over a year of development.  We released scanned document manager version 1.0.  This tool is now been in development for a number of years and supports many advanced features.  It stays true to its roots, however, and delivers a simple solution for automatically filing documents based on the client mentioned in the document, whether by client name or another.  Identifying item.

 

Scanned document manager runs as an application on a user's PC.  It does not run on Macs.  When running on a PC.  It runs in the background.  It can either wait for searchable PDFs to appear in a folder or allow the end-user to drop them onto the application.  Once a searchable PDF has been seen by scanned document manager.  It will be analysed and either automatically saved to the client folder or if set in the configuration, a pop-up will appear to allow the user to verify where the document will be filed.

 

At first, use scanned document manager does not always get the client name correct.  This can be due to a number of items, but most commonly is due to the name of the practice being a client in the practice management system.  One of the very simple things you can do to make scanned document manager highly effective out-of-the-box is, if the practice name is included as a client in the practice management system to add [internal] to the end of the night.  This will ensure less false-positive is and give a far better experience out-of-the-box.

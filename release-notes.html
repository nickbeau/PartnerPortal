A word on version  numbering:

             In our development process (MSF for Agile 6.0), we version as follows: (AA.bb.cccc) where AA is the major version number. CA Karios release will be 1.0. bb is the minor version number which gets incremented each release unless a major rewrite occurs. Cccc is the build number and increases everytime we compile the solution.

 

Release version will be 1.0.xxx and all components will adhere to that numbering on release.

 

All the below is tracked in visual studio online: https://hubone.visualstudio.com/DefaultCollection/CAANZ%20Kairos
